## 4.3 Оценка производительности и устойчивости развёрнутого приложения

**Наблюдения:**

- время загрузки главной страницы < 2 с (Tailwind CSS minified, статика через Express);
- API `GET /api/products` — ответ < 200 ms при < 100 товарах в SQLite;
- автоматический restart на Railway при push в GitHub;
- `ensureDb()` при старте — idempotent миграции, без потери данных при redeploy (при наличии Volume).

**Ограничения:** SQLite не предназначена для высокой concurrent-нагрузки; для production-масштаба рекомендуется PostgreSQL.

## 4.4 Охрана труда и безопасность при разработке программного продукта

Разработка веб-приложения связана с длительной работой за персональным компьютером. Соблюдались следующие требования охраны труда:

### Организация рабочего места

- монитор на расстоянии 50–70 см от глаз, верхний край на уровне глаз;
- кресло с поддержкой поясницы;
- освещение 300–500 lux, без бликов на экране;
- клавиатура и мышь на одном уровне локтей.

### Режим труда и отдыха

- перерыв 10–15 минут каждые 2 часа (гимnastika для глаз и разминка);
- соблюдение санитарных норм при работе не менее 4 часов в сутки за ПК.

### Электробезопасность

- использование исправного оборудования и заземлённых розеток;
- запрет на самостоятельный ремонт блоков питания;
- при работе с сервером Railway — только удалённый доступ, без высоковольтного оборудования.

### Пожарная безопасность

- не загромождать пути эвакуации;
- знать расположение огнетушителя;
- не оставлять включённую технику без присмотра.

## 4.5 Информационная безопасность веб-маркетплейса

### Аутентификация и хранение паролей

- пароли **не хранятся** в открытом виде;
- используется **bcrypt** (cost factor 10) — `server/routes/auth.js`;
- минимальная длина пароля — 6 символов.

### Сессии

- **express-session** с httpOnly cookie (защита от XSS-доступа к cookie из JS);
- `secure: true` в production (только HTTPS);
- `SESSION_SECRET` из переменных окружения Railway;
- срок сессии — 7 суток.

### Авторизация (RBAC)

| Роль | Доступ |
|------|--------|
| guest | Каталог, корзина, заказ, регистрация |
| client | + account, orders/mine |
| admin | + CRUD товаров, upload, orders/admin |

Middleware `requireAuth`, `requireAdmin` на сервере — клиентская проверка дублирует UX, но не заменяет серверную.

### Защита API

- admin-эндпоинты возвращают 401/403 без валидной сессии;
- валидация входных данных (телефон, обязательные поля заказа);
- SQL — parameterized queries (better-sqlite3 prepare), защита от SQL-injection;
- загрузка файлов: whitelist MIME (jpeg, png, webp), лимит 5 MB.

### Транспортный уровень

- Railway предоставляет HTTPS по умолчанию;
- `app.set('trust proxy', 1)` для корректных secure-cookie за reverse proxy.

### Рекомендации для production

1. Сменить `ADMIN_PASSWORD` и `SESSION_SECRET` на криптостойкие значения
2. Rate limiting на `/api/auth/login`
3. CSRF-токены для state-changing запросов
4. Регулярный backup SQLite (Volume snapshot)
5. Не коммитить `.env` в Git (файл в `.gitignore`)
