# 1 АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ И ПОСТАНОВКА ЗАДАЧИ

## 1.1 Современное состояние электронной коммерции и маркетплейсов в Казахстане

Электронная коммерция (e-commerce) в Казахстане развивается опережающими темпами. Ключевые факторы роста:

- высокая проникаемость смартфонов и мобильного интернета;
- развитие финтех-экосистемы (Kaspi, Halyk, Homebank);
- рост доверия населения к онлайн-покупкам после пандемии COVID-19;
- государственная поддержка цифровизации бизнеса.

**Маркетплейс** — это программная платформа, объединяющая множество продавцов и покупателей. Продавец размещает карточки товаров, покупатель выбирает товар, оформляет заказ и оплачивает его удобным способом. Администратор платформы управляет каталогом, заказами и пользователями.

Для казахстанского рынка характерны:

| Особенность | Описание |
|-------------|----------|
| Доминирование Kaspi | Более половины безналичных операций проходит через экосистему Kaspi |
| Мобильность | Большинство покупок совершается с мобильных устройств |
| Языковое разнообразие | Необходимость интерфейса на казахском, русском и часто английском языках |
| Региональность | Важность указания региона производства для отечественной продукции |

## 1.2 Анализ существующих решений

**Kaspi Market** — крупнейший маркетплейс Казахстана, интегрированный в супер-приложение Kaspi. Преимущества: огромная аудитория, встроенная оплата Kaspi, доставка. Недостатки для нишевого сегмента: высокая конкуренция, комиссии, сложность выхода малых производителей, отсутствие акцента на «только KZ-продукцию».

**Wildberries / Ozon** — международные и федеральные площадки с доставкой в РК. Широкий ассортимент, но не специализированы на казахстанском производстве.

**Социальные сети и Instagram-магазины** — часто используются малыми производителями, но не обеспечивают единый каталог, корзину, историю заказов и безопасную авторизацию.

**Вывод:** существует ниша для специализированного маркetplace с фокусом на отечественную продукцию, простой админкой для производителя и оплатой, привычной для казахстанского покупателя.

## 1.3 Проблемы продвижения отечественной продукции

1. **Фрагментированность каналов продаж** — производитель продаёт через WhatsApp, Instagram, ярмарки; нет единой витрины.
2. **Низкая узнаваемость** — покупатель не знает, где искать «Made in Kazakhstan».
3. **Отсутствие цифровой инфраструктуры** — малый бизнес не может позволить дорогую разработку.
4. **Языковой барьер** — часть аудитории предпочитает казахский интерфейс.
5. **Логистика и оплата** — нужны привычные способы: Kaspi-перевод, оплата при получении, самовывоз.

**QazMarket** решает эти проблемы в учебном/пилотном масштабе: единый каталог, регион и производитель на каждой карточке, три языка, Kaspi + COD, самовывоз с точкой на карте.

## 1.4 Обоснование выбора способов оплаты для рынка РК

При проектировании системы рассматривалась интеграция **Stripe** — международного платёжного агрегатора. Однако Stripe **не поддерживает** регистрацию merchant-аккаунтов для бизнеса, зарегистрированного в Казахстане. Для дипломного проекта, ориентированного на реальный рынок РК, выбраны:

| Способ | Описание | Обоснование |
|--------|----------|-------------|
| **Kaspi (перевод)** | После заказа покупатель переводит сумму на номер продавца в приложении Kaspi; комментарий — номер заказа | ~85% взрослого населения используют Kaspi; самый распространённый способ P2P-оплаты |
| **Kaspi QR** | QR-код со ссылкой на kaspi.kz для быстрого перехода в приложение | Удобство для мобильных пользователей |
| **Оплата при получении (COD)** | Оплата курьеру или при самовывозе | Привычный способ для регионов и старшего поколения |

Администратор подтверждает поступление Kaspi-перевода в панели заказов (`/admin-orders.html`).

## 1.5 Постановка задачи и требования к веб-приложению QazMarket

**Постановка задачи:** разработать веб-маркетплейс QazMarket для демонстрации продажи продукции казахстанского производства с полным циклом «каталог → корзина → заказ → оплата/получение» и административным управлением контентом.

### Функциональные требования

**Для покупателя (клиента):**
- просмотр каталога с фильтрами (категория, регион, цена, поиск);
- просмотр карточки товара (фото, описание, производитель, регион);
- корзина (localStorage);
- регистрация и вход;
- личный кабинет: профиль (фамилия, имя, телефон), история заказов;
- оформление заказа: доставка или самовывоз;
- выбор оплаты: Kaspi или при получении;
- интерфейс на kk / ru / en.

**Для администратора:**
- отдельный вход (без публичной регистрации);
- добавление, редактирование, удаление товаров;
- загрузка фото (авто-обрезка 800×800);
- просмотр заказов, подтверждение Kaspi-оплаты.

### Нефункциональные требования

- развёртывание в облаке (Railway) с постоянным URL;
- хранение данных в SQLite с автоматической инициализацией БД;
- хеширование паролей (bcrypt), сессии (express-session);
- адаптивная вёрстка (Tailwind CSS);
- время отклика API < 1 с при типовой нагрузке.

### Ограничения

- учебный проект: нет полноценной интеграции Kaspi Merchant API (используется сценарий перевода по номеру);
- SQLite вместо PostgreSQL (достаточно для демо и диплома);
- без мобильного нативного приложения (responsive web).
